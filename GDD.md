# Case by Case

# Overview

## Elevator Pitch

In this game, players evaluate small program functions by selecting a limited number of test cases from a larger pool. Each test case serves as evidence about the function's behavior, and players cannot submit every test; they must decide which evidence is most meaningful.

The game frames each function as being "on trial," where confidence in its behavior depends entirely on the quality of the evidence presented. Some functions work correctly but lack sufficient evidence to justify a strong verdict, while others appear broken due to misleading or flawed test cases.

Rather than using simple pass or fail outcomes, the game issues confidence-based verdicts, emphasizing that unit tests do not prove correctness, but instead establish confidence based on the evidence chosen. Through this process, players learn to evaluate test quality, recognize insufficient coverage, and reason about edge cases.

## Core Gameplay Mechanics (Detailed)

- Each function is presented as being "on trial."
- Players choose only a limited subset of test cases as evidence.
- Confidence in behavior is based on evidence quality, not proof of correctness.
- Outcomes are confidence-based verdicts rather than binary pass/fail.
- Players learn through identifying strong, weak, misleading, and insufficient evidence.

## Feedback

- Verdicts reflect confidence based on selected evidence.
- Stronger evidence supports stronger confidence in behavior.
- Misleading or flawed evidence can produce weak or incorrect conclusions.
